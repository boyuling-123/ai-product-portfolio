# 作品集补全清单

更新时间：2026-06-24

说明：这份清单只列当前作品集 HTML 还缺的内容、图片和发布前材料。已经有真实截图的内容不重复要求补图；不确定、未脱敏、无法证明的内容继续保留为空证据位，不用推测图冒充真实项目。

## 一、当前 HTML 明确缺少的图片和证据

### P0 必须优先补

- Brand+ 电商图片处理平台
  - 缺少内容：脱敏产品界面截图。
  - 建议素材：首页、工作台、功能列表、生成结果页截图 2-3 张。
  - 用途：替换首页里的“业务链路证据位 / 功能矩阵证据位 / 测评体系证据位”。
  - 格式建议：PNG / JPG。

- Brand+ 指标证明
  - 缺少内容：指标口径和数据证明。
  - 建议素材：统计周期、计算方式、数据来源、指标截图或表格。
  - 用途：支撑 1000W+、97%、96% 这类成果，避免作品集显得空泛。
  - 格式建议：表格截图 / Markdown 表格 / Excel 导出截图。

- 商品卖点图重构
  - 缺少内容：完整输入到输出案例。
  - 建议素材：商品原图、商品信息、卖点提取过程、Prompt、最终卖点图。
  - 用途：这是新整理出来的重点 skill，适合做成强案例。
  - 格式建议：Markdown + PNG / JPG。

- 联系方式
  - 缺少内容：个人公开联系信息。
  - 建议素材：邮箱、GitHub 总主页、个人站点、微信二维码可选。
  - 用途：补全“关于我 / 联系方式”区域。
  - 格式建议：文本 / 图片。

## 二、发布级作品集还缺的内容

### P0 发布前必须补

- 公开入口
  - 缺少内容：公网作品集地址。
  - 建议形式：Vercel / GitHub Pages / Cloudflare Pages / 个人域名。
  - 用途：让作品集可以直接发给别人看。

- 个人介绍
  - 缺少内容：正式个人简介。
  - 建议内容：一句话定位、核心能力、代表项目、当前方向。
  - 用途：补强“关于我”区域。

- 原创性说明
  - 缺少内容：哪些是原创、哪些是整理、哪些是外部维护 skill。
  - 建议内容：按项目或 skill 标注“原创主导 / 二次整理 / 外部维护 / 参考接入”。
  - 用途：避免别人误解 skill 来源，也保护你自己。

- 核心案例详情页
  - 缺少内容：Brand+、商品卖点图重构、测评平台的深度案例。
  - 建议内容：背景、问题、方案、流程、结果、反思、证据图。
  - 用途：让作品集不只是卡片合集，而是能体现产品思考。

### P1 发布前建议补

- PDF 简历版本
  - 缺少内容：可下载简历。
  - 建议内容：一页或两页 PDF，突出 AI 产品、AI Coding、Skill、测评平台。
  - 用途：方便投递和转发。

- SEO / 分享信息
  - 缺少内容：favicon、OG image、页面 title、description、分享封面。
  - 建议内容：个人名 + AI 产品作品集主题图。
  - 用途：发链接时更专业。

- 项目封面图统一风格
  - 缺少内容：统一比例的封面图。
  - 建议内容：每个重点项目一张 16:9 封面，带浏览器边框或产品标题。
  - 用途：让作品集视觉更整齐。

- 深色 / 浅色主题
  - 缺少内容：主题切换。
  - 建议内容：按钮切换并本地持久化。
  - 用途：提升完成度和观感。

### P2 锦上添花

- 404 页面
  - 缺少内容：自定义 404。
  - 用途：部署后更完整。

- 访问统计预留
  - 缺少内容：埋点结构。
  - 建议内容：先留空函数或配置位，不直接填第三方 key。
  - 用途：方便后续看访问数据。

- 项目截图优化
  - 缺少内容：浏览器边框 mockup。
  - 建议内容：给真实截图统一加边框、阴影和标题栏。
  - 用途：提高展示质感。

- 更多案例详情页
  - 缺少内容：PPC、虚拟穿戴、电商详情页、Prompt 管理平台详情页。
  - 用途：后续有素材后继续扩展。
