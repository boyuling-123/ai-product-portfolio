Business Case

堆友 Agent 建设

面向 AI 设计场景，参与堆友多 Agent 架构建设与图像生成编辑 Agent 方案设计，把图像生成、图像编辑、文本处理、任务规划、状态记忆和结果总结拆成可独立迭代的 Agent / Tool 组。

Multi-Agent图像生成/编辑Human-in-loop

项目描述

项目目标是提升 Agent 对复杂设计任务的理解、拆解、执行和复盘能力，支持多轮交互、上下文记忆、用户问询、模型路由、图像生成与编辑等能力。我的工作重点是把混在单一 Agent 里的能力拆成上层调度、图像任务、文本任务和底层 Tool，并明确每个节点的输入输出、路由条件和异常处理方式。

需求与设计

  • 设计 mainSupervisor 作为上层调度节点，负责用户意图理解、任务规划、完整性检查、任务路由和结果汇总。
  • 设计 imageSupervisor 负责图像任务判断与分发，根据用户 prompt、图片列表和任务意图选择 IMAGE_GEN、IMAGE_EDIT 或结束流程。
  • 设计 TextProcess 文本处理节点，抽象 EDIT、TRANSLATION、EXPANSION、SUMMARIZATION 等操作，支持文本修改、翻译、扩写和总结。
  • 设计 state 记忆结构，沉淀 userInput、taskStatusList、mainInstruction、imageInstruction、mainSummary、图片列表、任务状态和异常信息。
  • 针对用户输入不完整、意图不清晰或关键参数缺失的情况，设计 human-in-loop 问询节点，让 Agent 在执行前主动追问补充信息。

技术与执行

图像生成链路

梳理文生图、单图生图、二维码生成、Nano Banana 生成等能力的参数要求与路由边界。

图像编辑链路

梳理 userPrompt、imageUrl、width、height 等参数，并用 reason 字段记录模型选择原因。

模型与 Tool 接入

接入即梦、Qwen Image、Nano Banana、二维码生成、即梦编辑、Qwen Image Edit 等能力。

可观测与复盘

通过任务状态、节点结果、用户问询和模型选择原因，让 Agent 从黑盒调用变成可观察流程。

项目管理与优化

  • 推动 Agent 从“单一对话 + 工具调用”升级为“任务规划 + 多 Agent 协作 + 状态记忆 + 工具路由 + 用户问询”的结构化工作流。
  • 通过拆分 supervisor、imageSupervisor、TextProcess 和底层工具组，降低复杂设计任务的耦合度，使图像生成、图像编辑、文本处理、结果总结等能力能够独立迭代。
  • 结合模型评测知识库和不同模型擅长领域，探索在同一功能下接入多个生成/编辑模型，并根据任务类型和评测结果进行模型选择。
  • 为后续模型路由、bad-case 回流和 Agent 效果优化打基础，也能反哺 AI 测评平台的评估维度与样本沉淀。
