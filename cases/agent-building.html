Business Case

堆友 Agent 建设

面向 AI 设计场景，参与堆友多 Agent 架构建设与图像生成编辑 Agent 方案设计，把图像生成、图像编辑、文本处理、任务规划、状态记忆和结果总结拆成可独立迭代的 Agent / Tool 组。

Multi-Agent图像生成/编辑Human-in-loop

项目描述

项目目标是提升 Agent 对复杂设计任务的理解、拆解、执行和复盘能力，支持多轮交互、上下文记忆、用户问询、模型路由、图像生成与编辑等能力。我的工作重点是把混在单一 Agent 里的能力拆成上层调度、图像任务、文本任务和底层 Tool，并明确每个节点的输入输出、路由条件和异常处理方式。

需求与设计

  • 设计 mainSupervisor 作为上层调度节点，负责用户意图理解、任务规划、完整性检查、任务路由和结果汇总。
  • 设计 imageSupervisor 负责图像任务判断与分发，根据用户 prompt、图片列表和任务意图选择 IMAGE_GEN、IMAGE_EDIT 或结束流程。
  • 设计 TextProcess 文本处理节点，抽象 EDIT、TRANSLATION、EXPANSION、SUMMARIZATION 等操作，支持文本修改、翻译、扩写和总结。
  • 设计 state 记忆结构，沉淀 userInput、taskStatusList、mainInstruction、imageInstruction、mainSummary、图片列表、任务状态和异常信息。
  • 针对用户输入不完整、意图不清晰或关键参数缺失的情况，设计 human-in-loop 问询节点，让 Agent 在执行前主动追问补充信息。

技术与执行

图像生成链路

梳理文生图、单图生图、二维码生成、Nano Banana 生成等能力的参数要求与路由边界。

图像编辑链路

梳理 userPrompt、imageUrl、width、height 等参数，并用 reason 字段记录模型选择原因。

模型与 Tool 接入

接入即梦、Qwen Image、Nano Banana、二维码生成、即梦编辑、Qwen Image Edit 等能力。

可观测与复盘

通过任务状态、节点结果、用户问询和模型选择原因，让 Agent 从黑盒调用变成可观察流程。

项目管理与优化

  • 推动 Agent 从“单一对话 + 工具调用”升级为“任务规划 + 多 Agent 协作 + 状态记忆 + 工具路由 + 用户问询”的结构化工作流。
  • 通过拆分 supervisor、imageSupervisor、TextProcess 和底层工具组，降低复杂设计任务的耦合度，使图像生成、图像编辑、文本处理、结果总结等能力能够独立迭代。
  • 结合模型评测知识库和不同模型擅长领域，探索在同一功能下接入多个生成/编辑模型，并根据任务类型和评测结果进行模型选择。
  • 为后续模型路由、bad-case 回流和 Agent 效果优化打基础，也能反哺 AI 测评平台的评估维度与样本沉淀。

两类工作模式

Supervisor 模式

适合开放性高的任务。上层 supervisor 负责意图理解、任务规划、Agent 调用和结果整理；子 Agent / tools 只负责具体执行，并与 supervisor 通信。

Custom Workflow 模式

适合部分确定性的复杂任务。提前定义子 Agent 通信顺序，子 Agent 可相互通信，也可在关键节点直接与用户交互。

配套组件

  • Host：LLM，负责任务规划、决策和生成响应。
  • Tools：API / MCP / 工作流能力，用于生成、编辑、搜索等动作。
  • 知识库：提供模型能力、风格、规则和历史经验。
  • 记忆：共享用户输入、需求分析、任务规划、进度、Agent 状态、中间结果和 tool 结果。

本期功能范围

  • 将现有 tool 能力改造成子 Agent：海报生成、图像生成、图像编辑、抠图、文本生成/编辑和聊天机器人。
  • 在部分子 Agent 中接入多个模型，结合评测知识库选择适合模型。
  • 在部分子 Agent 中增加用户选择、确认和追问节点。
  • P1 引入搜索 tool、字体更改、效果调整、内容调整等操作编辑 Agent。

Agent 构建框架

  • 能力建设：建立和维护 tool 库，接入外部 API、内部 API 和设计师自主搭建的工作流。
  • Agent 基建：通过统一 API 网关对外服务，支持业务身份管理、资源分配和任务监控。
  • 自研 Agent：适用于强自主可控、复杂工作流编排的项目。
  • 自由 Agent：适用于中低复杂度、开放式工作流和快速搭建项目。
  • 应用搭建平台：支持 Agent 搭建、测试和试用。

多 Agent 节点流程

  • 主体结构为 LLM + 子 Agent / tool，supervisor 进行任务理解和路由。
  • 子图包括海报、图片生成、图片编辑等，每个子图执行后合并到 summary 节点。
  • 记忆模块记录任务信息、任务规划、任务进度、Agent 状态、tool 入参和每轮结果。
  • 当意图不准确或输入不完整时，通过用户干预节点追问，提升任务规划准确度。

子 Agent / Tool 范围

  • P0：文本生成与处理、图像生成与编辑、文生图、单图编辑、多图编辑、抠图。
  • P1：视频生成、海报生成、分层海报、协议编辑、作品编辑、内容编辑。
  • P2：基于开放搭建平台的设计类 Agent 扩展。

视频能力建设

  • 视频编辑 Agent 目标：让 AI 通过代码生成视频描述或视频代码，再渲染到编辑器中。
  • 第一阶段实现本地渲染组件编辑器化，打通编辑器交互与协议双向互通。
  • 第二阶段基于现有渲染方案扩展自然语言编辑、元素编辑和效果增强。
  • 视频 Agent 产品形态：在视频编辑器左侧加入 Agent 对话窗口，并构建视频系统 Prompt 与工具封装。

核心架构与模块

  • 能力模块：Skill 载入、tool 分层管理、Prompt 管理与调试。
  • 记忆模块：上下文优化、长期信息保留和任务状态复用。
  • 中间件：消息规范、通信机制和运行时事件。
  • 运行时：双层循环处理用户追问、tool calls 和用户打断；事件系统记录生命周期。
