Business Case

悟空 MCP 对接

围绕业务 Agent 如何接入超编 / 悟空类平台，梳理 API、MCP、Skill、ComfyCloud 工作流、Oneday Agent 和 LangGraph 子 Agent 的能力边界，让业务能力能被平台调用、管理、观察和复用。

MCP / SkillAgent 平台接入能力网关

项目描述

平台对接的核心不是把某个接口“接通”，而是定义业务垂直场景 Agent 如何进入平台生态：哪些能力作为 Tool，哪些能力作为子 Agent，哪些能力沉淀为 Skill，过程数据如何透出，结果如何回写，失败时如何兜底。

能力层外部 API、内部 API、ComfyCloud 工作流 API、Oneday Agent API、LangGraph 子 Agent。
平台层MCP / Tool Schema、业务身份、资源分配、流量管控、任务监控、日志和错误码。
业务层垂直场景 Agent、灵动岛对话框、业务定制组件、画布结果编辑和业务系统回传。

需求与设计

  • 梳理 API、MCP、Skill、Workflow、Agent 的能力层级：简单能力以 Tool 形式接入，复杂链路以 Skill / 子 Agent 形式承接。
  • 定义接入平台的标准信息：能力名称、使用场景、输入参数、输出协议、调用方式、权限边界、状态事件和错误处理。
  • 区分 Oneday Agent 与 LangGraph 子 Agent 的适用边界：前者适合快速搭建和 API 化调用，后者适合复杂工作流、自主可控和过程交互。
  • 规划平台网关能力：业务身份管理、资源分配、流量管控、任务监控、日志追踪和失败兜底，避免能力散落在各业务侧。

技术与执行

Tool Schema

整理每个能力的入参出参、必填字段、枚举值、默认值、返回结构和错误码。

过程输出

明确 Agent 执行过程中的状态事件、卡片渲染、流式输出和用户确认节点。

业务定制

区分平台通用交互与业务定制逻辑，支持会员、扣费、素材库、业务内容中心等能力接入。

验收清单

沉淀成功样例、失败样例、日志字段、重试口径和人工兜底标准，便于跨团队联调。

项目管理与优化

  • 把“能力能不能调通”上升为“能力能不能被平台稳定管理和复用”，推动能力接入标准化。
  • 协同产品、研发、设计和业务侧确认接入范围、交互边界、排期和验收标准。
  • 为后续视频生成 Skill、PRD Skill、买点图生成、模特图生成、详情页生成等能力接入提供统一口径。

业务垂直场景 Agent 接入与定制方案
Agent 搭建平台业务定制能力网关过程输出

核心问题

不同业务的垂直场景 Agent 需要接入统一设计编辑器，并在编辑器中完成状态交互、结果展示、画布编辑、业务逻辑定制和结果回写。方案需要同时解决能力搭建与业务交互两类问题。

能力构建层

  • 接入外部 API、内部 API、工作流 API。
  • 支持低复杂度 Agent 以 API tool 形式接入。
  • 支持复杂工作流以自研子 Agent 形式接入。

平台接入层

  • 业务通过 Agent 搭建平台构建垂直场景 Agent。
  • 统一适配过程输出、交互数据和前端消息渲染。
  • 同一编辑器实例一次只接入一个业务 Agent，避免上下文混乱。

Agent 形态选择

  • 工作流平台 Agent：适合快速搭建、中低复杂度任务和 API 化调用；作为业务 Agent 中的 tool 使用。
  • 自研 LangGraph 类子 Agent：适合复杂工作流、自主可控、过程交互和深度适配；作为业务 Agent 中的子 Agent 使用。
  • Agent 应用搭建平台：作为能力 Agent 和编辑器之间的中间层，负责下游能力接入和上游消息适配。

业务交互定制

  • 对话入口提供通用交互框架和自适应渲染组件。
  • 业务可在通用框架基础上定制卡片、步骤、确认节点、扣费/权限/素材库等业务逻辑。
  • 画布区域保持平台通用能力，业务只控制 AI 能力、素材内容、业务组件和结果回传。
  • 业务定制 Agent 仅通过能力层与编辑器通信，避免业务逻辑直接侵入底层编辑器。

一期开发范围

  • 以一个典型海报/设计业务 Agent 为试点，打通能力服务、搭建平台、对话入口、画布结果和业务回传。
  • Agent 能力侧完成通用图像生成/编辑、提示词优化、用户干预逻辑、分层海报与协议编辑。
  • 编辑器侧完成 Agent 协议、基础卡片、通信 SDK、对话入口、标注/修改和业务开发规范。
  • 搭建平台侧完成 Agent Response 数据结构、消息卡片渲染规范、Agent 消息流程和工具接入管理。

验收口径

  • 业务 Agent 能被平台调用，并能稳定输出状态、卡片、结果和错误信息。
  • 用户能在关键节点确认、修改或中断任务。
  • 生成结果能进入画布继续编辑，并能回传业务系统或项目资产。
  • 能力、工具、Agent 与业务定制组件之间有清晰边界，后续业务可复用同一接入口径。
项目需求池与迭代计划
需求池P0/P1/P2视频能力图片能力

管理目标

把视频、图片和业务平台侧的需求集中到统一需求池，用优先级、PRD 状态、设计状态、开发状态和发布状态管理跨团队迭代。

视频方向需求

  • P0：二次编辑交互、任务数据查询与存储、重新生成、失败交互、删除/清空、多个视频片段同时重跑。
  • P0：模板更新、模板封面、模板时长透出、主任务生成后的二次编辑和失败子任务透出。
  • P1：品牌或 logo 首尾帧、生成时长配置、音乐替换、更多分镜图生成能力、参考图生成分镜图、视频旁白。
  • P2：输出尺寸逻辑、AI 音乐生成、多语言批量生成、历史记录分页和性能优化。

图片方向需求

  • P0：手持商品图效果优化、虚拟试穿、白底图生成改造、图翻改造、详情页生成、商品主图生成。
  • P0/P1：模特换肤、图像编辑模型接入、单图/多图/自定义形式、模板优化和交互界面优化。
  • 平台侧依赖：自定义上传组件压缩规则、编辑器历史记录、工具接入规范和业务上架流程。

项目化沉淀

  • 需求池不仅记录功能点，还记录 PRD 是否完成、设计是否评审、开发是否提测、线上是否发布。
  • 对已发布能力保留状态；对待排期能力保留依赖、风险和下一步。
  • 将视频工具、图片工具和 Agent 改造放在同一个池子里，便于识别跨工具依赖。
