Business Case

Brand+ 电商图片处理平台

面向电商业务的内部图片处理平台，由 AI 设计中台提供底层 AI 能力支持。我负责把白底图、图片翻译、详情页多语种和卖点图等场景拆成可调用、可测评、可复用的产品能力。

电商图片白底图 / 图翻 / 详情页测评闭环

项目描述

Brand+ 的关键问题不是“有没有模型”，而是底层图像能力如何稳定进入真实电商生产链路。项目需要同时处理业务输入、工具唤起、生成结果、二次编辑、质量评估、历史任务和结果回传，最终支撑白底主图、多语言图片翻译、详情页处理、主图套图和卖点图等核心场景。

需求与设计

  • 白底图：设计图片上传、参数预填、白底图检测、通用抠图、背景色/尺寸设置、结果展示和继续编辑链路。业务唤起 iframe 时，可携带图片 URL、抠图参数和自动执行状态。
  • 图片翻译：拆解图片内容检测、语种判断、OCR / mask、图片消除、多语言翻译、内容诊断、AI 排版和协议转换，支持原语言与目标语言不同的翻译链路。
  • 详情页 / 卖点图：设计商品主图、已有详情图、PC / Mobile 详情图、卖点选择、参数补充、多语言生成和结果回传方式，把生成结果以画板形式进入编辑器。
  • 二次编辑：在超编底层能力上叠加业务定制功能，支持手动抠图、拼图、预设样式、配件图上传、图片优化和保存回传。

技术与执行

能力封装

将白底图检测、抠图、图翻、详情页生成等能力拆成清晰输入输出，便于业务平台通过 iframe / API 接入。

测评体系

围绕白底图、图翻、详情页等场景建立批量测试口径，覆盖生成质量、翻译质量、处理耗时和 bad-case。

协议链路

梳理图翻 JSON 协议、HTML 协议、超编协议之间的转换关系，保证生成结果可继续编辑。

Skill 化复用

把套图、详情页、PRD、会议纪要等能力沉淀为可复用 Skill，方便后续跨场景调用。

项目管理与优化

  • 对接业务、设计、算法和研发，拆解 P0 / P1 能力范围，明确白底图、图翻、详情页、卖点图等能力的开发优先级。
  • 结合简历中的项目口径，累计支撑 1000W+ 图片处理量；抠图准确率达 97%，多语言翻译优质率达 96%。
  • 将一次性模型调试经验沉淀为评测规则、Prompt、bad-case 和能力文档，降低后续能力接入成本。

Brand+ 图片能力需求
图片检测图片生成iframe 接入协议回传

目标与边界

把电商图片处理拆成可被业务平台调用的标准能力，包括图片文本内容检测、白底图检测、多语言内容诊断、白底图生成、图片翻译、详情页/卖点图生成和编辑工具接入。

检测能力

  • 图片文本内容检测：输入图片 URL，输出语种与字符数量，用于判断是否需要翻译或内容处理。
  • 白底图检测：输入图片 URL，输出是否符合白底规范及不符合原因。
  • 多语言内容诊断：输入图片或文本，输出语言质量评分与修改建议。

生成能力

  • 白底图：当商家图片不是白底时，调用抠图/白底能力生成合规主图。
  • 图片翻译：基于原语言、目标语言和图片 URL 生成翻译结果图及可编辑协议。
  • 详情页/卖点图：基于商品主图、详情图、语言参数和卖点选择生成新的营销图片。

核心流程

  • 白底图链路：业务传入主图和参数，工具预填尺寸、背景色与抠图配置；生成后支持结果展示、历史记录与继续编辑。
  • 图片翻译链路：先识别图片内容与语种，再进行 OCR / mask / 消除 / 翻译 / 诊断 / AI 排版，最后输出可继续编辑的图片协议。
  • 主图套图/卖点图链路：根据商品主图、详情图、类目和卖点，判断是否需要翻译、补图、重排版或生成新卖点图。
  • 工具接入链路：能力通过 iframe 方式进入业务平台，支持请求鉴权、任务状态消息、结果 URL 与画布协议回传。

字段与接口口径

  • 通用字段：图片 URL、原语言、目标语言、商品 URL、详情图列表、背景色、尺寸、抠图参数、画布协议。
  • 图翻路由：接收原语言、目标语言和图片 URL，返回图片结果与可编辑协议。
  • 详情页路由：接收商品类目、商品主图、PC / App 详情图、源语言与目标语言。
  • 白底图路由：接收主图、是否需要抠图、背景色和二次编辑画布协议。

验收关注

  • 白底判断是否稳定给出符合/不符合及原因。
  • 图翻结果是否保留主体、logo 等不应翻译内容。
  • 详情页/卖点图生成是否能进入编辑器继续修改。
  • 业务接入后，任务状态、历史记录和结果回传是否完整。
Brand+ 工具迭代与能力建设
白底图改造图翻 Pro主图套图能力建设

迭代范围

围绕白底图、图片翻译、主图套图/详描生成三类工具做产品化改造，让工具既能独立使用，也能被业务平台带参数唤起并自动执行。

白底图生成

  • 输入页支持批量上传、尺寸设置、常用尺寸、自定义尺寸、背景色、常用色和自定义色盘。
  • 业务 iframe 唤起时可传入图片 URL、抠图参数和自动执行开关；单任务进入结果页，批量任务进入任务列表。
  • 结果页按任务时间展示，支持继续编辑；若已有二次编辑作品则进入历史作品，否则重新抠图并带入画布。
  • 编辑页基于通用画布能力叠加业务定制：手动抠图、拼图、预设样式、上传配件图、图片优化、保存回传。

图片翻译

  • 图翻 iframe 支持多个翻译结果展示和切换编辑，并允许业务隐藏左侧参数栏。
  • 能力层拆为图片内容检测、多语言诊断、图片翻译、图片消除、翻译润色、AI 排版和协议转换。
  • 同语言处理作为扩展链路评估：通过图翻、文本替换和 AI 排版解决图片内同语种文案优化问题。

主图套图 / 详描生成

  • 左侧功能栏包含商品主图上传、卖点选择、自定义卖点和参数补充。
  • 卖点与参数一一对应，按行业类目预设可选卖点，并允许用户补充文案和商品信息。
  • 画布采用多画板方案，生成后可继续编辑，但新增画板、画板数量和业务数据回传需要受控。

能力建设与验收

  • P0 能力包括白底图检测、通用抠图、业务定制抠图、交互式抠图、图片内容检测和多语言诊断。
  • P1 能力包括 AI 组合、抠图模型优化、图片优化、视角调整和商品精修。
  • 每个能力需要明确接口状态、批量测试样本、效果验证标准和业务规则。
