Business Case

悟空 MCP 对接

围绕业务 Agent 如何接入超编 / 悟空类平台，梳理 API、MCP、Skill、ComfyCloud 工作流、Oneday Agent 和 LangGraph 子 Agent 的能力边界，让业务能力能被平台调用、管理、观察和复用。

MCP / SkillAgent 平台接入能力网关

项目描述

平台对接的核心不是把某个接口“接通”，而是定义业务垂直场景 Agent 如何进入平台生态：哪些能力作为 Tool，哪些能力作为子 Agent，哪些能力沉淀为 Skill，过程数据如何透出，结果如何回写，失败时如何兜底。

能力层外部 API、内部 API、ComfyCloud 工作流 API、Oneday Agent API、LangGraph 子 Agent。
平台层MCP / Tool Schema、业务身份、资源分配、流量管控、任务监控、日志和错误码。
业务层垂直场景 Agent、灵动岛对话框、业务定制组件、画布结果编辑和业务系统回传。

需求与设计

  • 梳理 API、MCP、Skill、Workflow、Agent 的能力层级：简单能力以 Tool 形式接入，复杂链路以 Skill / 子 Agent 形式承接。
  • 定义接入平台的标准信息：能力名称、使用场景、输入参数、输出协议、调用方式、权限边界、状态事件和错误处理。
  • 区分 Oneday Agent 与 LangGraph 子 Agent 的适用边界：前者适合快速搭建和 API 化调用，后者适合复杂工作流、自主可控和过程交互。
  • 规划平台网关能力：业务身份管理、资源分配、流量管控、任务监控、日志追踪和失败兜底，避免能力散落在各业务侧。

技术与执行

Tool Schema

整理每个能力的入参出参、必填字段、枚举值、默认值、返回结构和错误码。

过程输出

明确 Agent 执行过程中的状态事件、卡片渲染、流式输出和用户确认节点。

业务定制

区分平台通用交互与业务定制逻辑，支持会员、扣费、素材库、业务内容中心等能力接入。

验收清单

沉淀成功样例、失败样例、日志字段、重试口径和人工兜底标准，便于跨团队联调。

项目管理与优化

  • 把“能力能不能调通”上升为“能力能不能被平台稳定管理和复用”，推动能力接入标准化。
  • 协同产品、研发、设计和业务侧确认接入范围、交互边界、排期和验收标准。
  • 为后续视频生成 Skill、PRD Skill、买点图生成、模特图生成、详情页生成等能力接入提供统一口径。
